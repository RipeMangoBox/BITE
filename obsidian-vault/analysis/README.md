# Analysis Notes

Structured paper analysis notes are written here.

Expected layout:

```text
obsidian-vault/analysis/<Category>/<Venue_Year>/<Year>_<Short_Title>.md
```

Each note is designed to be useful to both humans and agents: metadata,
problem framing, method summary, evidence anchors, formulas, tables, figures,
experiments, limitations, and comparison hooks.

Typical flow:

1. Place PDFs under `obsidian-vault/paperPDFs/`.
2. Analyze papers through `research-workflow`, `papers-batch-analyze`, or `scripts/run_local_paper_analysis.py`.
3. Rebuild indexes with `papers-build-index`.

Generated notes are personal research artifacts. Commit only curated examples
that you intentionally want to publish.
